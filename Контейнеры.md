Одна из ключевых функций контейреного движка заключается в том, чтобы обеспечить всем выполняемым контейнерам совместный доступ к одному и тому же ядру ОС. Эта конфигурация гораздо менее ресурсоемкая, чем модель VM, где каждому экземпляру предоставляется полная копия ОС [[Список литературы#^eb6c16]]<c. 472>.

У контейнера есть своя файловая система, и в нем выполняется как минимум один процесс. 

NB! В языке Java используется _исключительно_ ==передача по значению== -- в базовом языке нет механизмов передачи по ссылке или по имени [[Список литературы#^eb6c16]]<c. 584>

NB! В Java нет встроенной поддержки каррирования [[Список литературы#^eb6c16]]<c. 588>

NB! В системе типов Java нет единого корня (то есть нет общего надтипа у `Object` и `int`). Из-за этого на Java невозможно написать `List<int>`, что в результате приводит к автоупаковке и сопуствующим проблемам. Многие разработчики жалуются на то, что параметры-типы обобщенных типов стираются во время компиляции, ==но на самом деле проблемы с обобщениями в коллекциях чаще возникают из-за того, что в системе типов нет единого корня== [[Список литературы#^eb6c16]]<c. 588>
