==NB! В Java все объекты существуют в куче Java, а операции с ними выполняются только по ссылке [[Список литературы#^eb6c16]]<c. 651>==

Во внутренней реализации JVM хранит так называемую _v-таблицу_ (для каждого класса) с определением методов, соответствующих этому типу (в C++ аналогичная структура обычно называется _таблицей виртуальных функций_)

В HotSpot различные внутренние структуры данных, которые находятся в куче, обозначают общим термином oop (ordinary object pointer).

У каждого объекта Java должен быть заголовок объекта, которые содержит две разновидности метаданных:
- метаданные, которые относятся к конкретному экземпляру класса (mark-слово),
- метаданные, общие для всех экземпляров класса (klass-слово)

Структура заголовка объекта JVM начинается с mark-слова, за которым немедленно следует klass-слово. Таким образом, чтобы найти метод для выполнения, JVM переходит по указателю в метапространство, где она по v-таблице из klass определяет, какой именно код нужно выполнять.
Если в klass нет определения для метода, то JVM переходит по указателю к klass, который соответствует непосредственному надклассу, и делает повторную попытку. На этом процессе основана перегрузка методов в JVM [[Список литературы#^eb6c16]]<c. 653>.
