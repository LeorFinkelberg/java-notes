
Запечатанные типы (sealed types) -- это механизм Java (Java 17+), который позволяет ограничивать иерархию наследования: мы явно указываем какие классы могут наследовать данный класс или реализовывать данный интерфейс.

NB! То есть вместо того чтобы любой класс мог расширять ваш класс, вы перечисляет разрешенные подтипы.
==NB! Разрешенные подтипы должны быть `final`, `sealed` или `non-sealed`==. Вы контролируете иерархию наследования. Никто вне вашего модуля не сможет создать новый подтип
```java
// Запечатанный класс
public sealed class Shape 
    permits Circle, Rectangle, Triangle {
}

// Ошибка компиляции! Не разрешен в permits
public class Square extends Shape { 
    // ...
}
```

Запечатанные типы отлично работают с pattern matching (Java 17+)
```java
// Компилятор знает, что есть только три возможных типа
double area = switch (shape) {
    case Circle c -> Math.PI * c.getRadius() * c.getRadius();
    case Rectangle r -> r.getWidth() * r.getHeight();
    case Triangle t -> t.getBase() * t.getHeight() / 2;
    // Не нужен default - компилятор знает, что все варианты покрыты!
};
```

Разрешенные подтипы могут быть:
- `final` -- больше нельзя наследовать
```java
public final class Circle extends Shape { }
```
- `sealed` -- иерахия продолжается
```java
public sealed class Polygon extends Shape 
    permits Quadrilateral, Triangle {
}
```
- `non-sealed` -- открыть для произвольного наследования
```java
public non-sealed class Rectangle extends Shape {
    // Любой класс может наследовать Rectangle
}
```

Классы могут объявлять пакетно-доступный (package-private) конструктор, что фактически означает "может расширяться только другими членами того же пакета". 

Пример абстрактного запечатанного класса
```java
public abstract sealed class Pet {
	private final String name;
	
	protected Pet(String name) {
	  this.name = name;
	}
	
	public String name() {
	  return name;
	}
	
	public static final class Cat extends Pet { // FINAL!
	  public Cat(String name) {
	    super(name);
	  }
	  
	  void meow() {
	    System.out.println(name() + "meow");
	  }
	}
	
	public static final class Dog extends Pet { // FINAL!
	  public Dog(String name) {
	    super(name);
	  }
	  
	  void bark() {
	    System.out.println(name() + "bark");
	  }
	}
}
```

Без уточнений `sealed` означает, что класс можно расширять только внутри текущей единицы компиляции, а следовательно, подклассы должны быть вложены в текущий класс. Класс `Pet` также объявлен абстрактным, потому что нам не нужны дополнительные экземпляры `Pet` помимо объектов `Pet.Cat` и `Pet.Dog`.

Интерфейсы тоже можно запечатывать.

Пример
```java
// ЗАПЕЧАТАННЫЙ интерфейс
public sealed interface FXOrder permits MarketOrder, LimitOrder {
    int units();
    CurrencyPair pair();
    Side side();
    LocalDateTime sentAt();
}

public record MarketOrder(
    int units,
    CurrencyPair pair,
	Side side,
	LocalDateTime sentAt,
	boolean allOrNothing
) implements FXOrder {
  // ...
}

public record LimitOrder(
    int units,
    CurrencyPair pair,
    Side side,
    LocalDateTime sentAt,
    double price,
    int ttl
) implements FXOrder {
  // ...
}
```

Новое ключевое слово `permits` позволяет разработчику перечислить все допустимые реализации этого запечатанного интерфейса, а наши реализации -- это записи.

==NB! Если используется `permits`, реализующие классы не обязательно размешать в том же файле: они могут быть отдельными единицами компиляции==

Поскольку `MarketOrder` и `LimitOrder` являются полноценными классами, у каждого из них может быть поведение, специфичное для его типа. 

_Запечатанные типы_ добавляют концепцию, которая раньше не моделировалась в Java: "Тип $X$ является либо $Y$, либо $Z$" [[Список литературы#^eb6c16]]<c. 116>

В терминах теории объектно-ориентированного программирования _запечатанные типы_ представляют новую разновидность формальных отношений, потому что набор возможных типов является объединением $Y$ и $Z$. Соответсвенно, во многих языках это называется _типом-объединением_ (union type) или _типом-суммой_ (sum type), но это совсем не то же самое, что `union` в языке С [[Список литературы#^eb6c16]]<c. 117>.
